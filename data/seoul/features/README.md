# data/seoul/features 데이터 사전 (X 피처)

> 좌표/ARS 기반 도시구조 피처를 소스별 1파일로 누적. 최종 단계에서 ARS·좌표로 모델 테이블에 조인.
> 공통 키 = ARS(텍스트, leading zero 보존). 대상 = 모델용 2,555 정류장.

## 0. `정류장_좌표.csv` ⭐ 좌표 베이스 (모든 공간 피처의 토대)
- 모델 2,555 정류장 + 마스터 좌표(ARS 조인). **2,553 좌표 보유 / 결측 2**(03726·22399 — 마스터에 없음).
- 컬럼: `ARS`, `자치구`, `정류장이름`, `경도`, `위도`. (EPSG:4326)
- 빌드: `uv run python scripts/build_stop_coords.py`

## 1. `승하차_노선_3-4월.csv` ⭐ 우선순위 1-A,B (승하차 + 경유 노선 수)
- **출처**: 열린데이터광장 [OA-12912](https://data.seoul.go.kr/dataList/OA-12912/S/1/datasetView.do). 월별 CSV `BUS_STATION_BOARDING_MONTH_202603/202604.csv`(cp949), `data/seoul/raw/`.
- 조인 키 = ARS. 매칭 2,479 / 결측 76(3.0%).

| 컬럼 | 설명 |
|---|---|
| `승차_일평균`,`하차_일평균` | (두 달 승/하차 합)÷(두 달 고유 일수). 명/일 |
| `승하차_일평균` | 승차+하차. **주 피처(유동 프록시)** |
| `경유노선수` | 두 달 고유 `노선번호` 수(합집합) |

- 분포: 승하차_일평균 중앙 848(p10 208~p90 2,747, max 11,768) · 경유노선수 중앙 4(max 21).
- ⚠️ **결측 76 = ✈️ 공항버스 54 + 경계/광역버스 22**(시내버스 카드데이터 부재, 키 버그 아님). **NaN 유지 권고**(0 대입 시 고가입지 왜곡, XGBoost는 NaN 처리).

## 2. `공시지가.csv` ⭐ 우선순위 2 (개별공시지가)
- **출처**: 국토부 **V-World 데이터 API**(`LP_PA_CBND_BUBUN` 연속지적도). 좌표 point 질의 → 속성 `jiga` 직접 수집(별도 PNU 조회 불필요). 키=`.env`.
- **2,553 좌표 전수, 필지 매칭 100%**. 공시지가>0 = 2,530(99.1%), 0/null 23개(도로 등 특수필지).

| 컬럼 | 설명 |
|---|---|
| `공시지가_원per㎡` | 개별공시지가(원/㎡). **주 피처(입지 토지가치)** |
| `PNU`,`지번`,`공시지가_주소`,`공시연도` | 필지 식별·추적 |

- 분포(원/㎡): min 30,700 · p10 1,044,700 · **중앙 3,142,000** · p90 13,720,900 · max 141,600,000.
- ⚠️ **단일점(point) 방식**: GPS 점이 떨어진 필지의 값. 중앙차로=도로필지여도 공시지가 보유(매칭 100%). 점 위치 노이즈는 향후 **반경 50m 평균**으로 보강 가능(현재 MVP 단일점). jiga=0인 23개는 NaN 취급 검토.
- 빌드: `uv run python scripts/build_jiga_feature.py [N]`  (N=샘플, 무인자=전체)

## 3. `POI_상권.csv` ⭐ 우선순위 3 (상권 POI 밀도)
- **출처**: 소상공인 상가정보 서울 202603 (**537,489 업소**, 경위도+업종). 로컬 KDTree 계산(**API 호출 0**). 출처상세 `../raw/README.md`.
- 등거리 투영(서울 중심) 후 반경 질의.
- 컬럼(13): `POI_total_{100,300,500}m`(전체 밀도 그래디언트) + `POI_{대분류}_300m`(도보권 업종 구성, 10종).
  - 대분류: 음식·소매·과학·기술·수리·개인·교육·부동산·시설관리·임대·예술·스포츠·보건의료·숙박.
- 분포: total 100m 중앙 42 / 300m 중앙 325 / 500m 중앙 900. **검증: 최밀 강남·교대역 300m 3,500+개.**
- ⚠️ '상가' 위주(지하철역·학교 등 비상업 POI 제외 — 1-C 지하철은 별도). 향후 필요 시 카카오로 앵커 보강.

## 4. `중심지거리.csv` ⭐ 우선순위 5 (도심 접근성)
- 다운로드·키 불필요. 좌표만으로 haversine 계산.
- 서울 **3도심**(2030 서울도시기본계획): 시청(한양도성)·강남·여의도.
- 컬럼: `거리_시청_km`,`거리_강남_km`,`거리_여의도_km`,`거리_도심최근접_km`.
- 분포(최근접 도심 km): **중앙 6.0**(p10 2.1~p90 11.0, max 15.3). 검증: 시청·강남역 0.1km, 도봉 수락 15km.

## 5. `지하철근접.csv` ⭐ 우선순위 1-C (지하철 접근성 + 역 이용객)
- **출처**: 서울 열린데이터광장 API(키 `.env` `SEOUL_OPENAPI_KEY`) — `subwayStationMaster`(역 783, 좌표) + `CardSubwayStatsNew`(역 승하차 3~4월 61일).
- 컬럼: `최근접지하철_거리_m`, `최근접역명`, `최근접역호선`, `최근접역_승하차_일평균`.
- 거리: KDTree 최근접(등거리 투영). 승하차: 역명 정규화 매칭(좌표역 645/783), **정류장 결측 54**(최근접이 경기구간 역/명칭 불일치) → NaN.
- 분포: 거리 **중앙 441m**(7~3,158). 최근접역 승하차 중앙 30,735명/일(max 서울역 244,600).
- 검증: 거리 **ρ=-0.429**(가까울수록 비쌈, 강한 신호) · 역 승하차 +0.276.

## 6. `도로망.csv` ⭐ 우선순위 4·6 (도로망 구조)
- **출처**: OpenStreetMap (osmnx 2.1, drive 네트워크). 키·다운로드 불필요 — 서울 정류장 bbox+~2km 버퍼를 **1회 fetch** 후 로컬 계산. 캐시 `../raw/osm_seoul_drive.graphml`(노드 86,397·무방향엣지 130,023·총연장 11,979km). 라이선스 **ODbL** → `../raw/README.md`.
- 미터 계산 **EPSG:5179** 투영. 엣지를 ~20m 점 샘플(가중치=구간길이) → KDTree 반경질의(기존 POI와 동일 패턴).

| 컬럼 | 설명 |
|---|---|
| `교차로수_{300,500}m` | 반경 내 교차로(차로 3갈래+ 노드) 수. **가로 결(미세 그리드)** 프록시 |
| `가로망연장_{300,500}m` | 반경 내 도로 총연장(m). 가로망 밀도 |
| `간선도로연장_{300,500}m` | 반경 내 간선도로(motorway/trunk/primary/secondary) 연장(m). **대로 노출** |
| `간선도로거리_m` | 최근접 간선도로까지 거리(m). 대로 접근성(반경 무관) |

- 분포(300m): 교차로수 중앙 33(p10 8~p90 82, max 170) · 가로망연장 중앙 5,103m · 간선도로연장 중앙 1,452m · 간선도로거리 **중앙 15m**(거의 전 정류장이 간선변).
- 검증(Y Spearman): 간선도로연장_300m **+0.191**(대로변일수록 비쌈, 기대방향) · 가로망연장 +0.136 · 교차로수 +0.118(약함) · 간선도로거리 **-0.024**(≈0).
  - ⚠️ **교차로수는 약한 신호**: 강남=슈퍼블록(교차로 적음·고가), 강북 구도심 골목=미세그리드(교차로 최다·저가, 삼양동 170개) → 순위상관 상쇄. **간선도로연장**(대로 노출)이 가격을 더 잘 잡음. 둘은 **상보적 도시형태 축** → XGBoost 상호작용·SHAP 재료.
  - ⚠️ **간선도로거리 ≈ 무신호**: 광고 정류장 중앙 15m로 거의 전부 간선변(버스가 간선 운행) → 분산 부족. 모델엔 두되 기여 낮을 것.
- 빌드: `uv run python scripts/build_road_feature.py` (첫 실행만 OSM fetch, 이후 캐시 재사용)
