# web/ — 지도 기반 광고가 조회 UI (웹 담당 작업 영역)

> 이 폴더는 **웹 프론트엔드 담당이 자유롭게 작업**하는 공간입니다.
> ML/모델 쪽(`../model`, `../data`, `../scripts`)은 **읽기만** 하고 수정하지 마세요(다른 팀원 영역).
> 전체 맥락: 루트 `../README.md`·`../CLAUDE.md`, 모델 상세: `../model/README.md`.

## 목표
지도에서 위치를 클릭(또는 주소/좌표 검색)하면, 그 지점 버스정류장의 **예측 광고가 + 오차범위 + "왜 그 가격인지"(SHAP 상위 요인)** 를 보여주는 웹.

## 핵심 결정: 모드 A — 격자 조회 (백엔드 API 불필요) ✅
- 예측은 이미 **`../model/grid_250m.csv`(서울 7,883칸, 250m 격자)** 에 전부 계산돼 있습니다.
- 웹이 할 일 = **"클릭 좌표에서 가장 가까운 격자 칸을 찾아 그 행을 표시"**. 서버·Python·모델 **불필요** → **정적 사이트로 충분**.
- (임의 좌표 실시간 계산이 꼭 필요해지면 그때 `../model/predict.py`를 Python API로 감싸는 모드 B를 논의. **지금은 범위 밖.**)

## 데이터 계약 — `../model/grid_250m.csv`
1행 = 격자 칸 1개. 주요 열:

| 열 | 의미 | UI 활용 |
|---|---|---|
| `위도`, `경도` | 칸 중심 좌표 | **최근접 칸 탐색 키** |
| `예측가_원per면` | 예측 광고가(원/면) | 메인 숫자 |
| `하한_원`, `상한_원` | 80% 예측구간 | "약 X원 (Y~Z)" |
| `top1_피처`~`top5_피처` / `top1_효과%`~`top5_효과%` | 위치별 SHAP 요인 (%=가격 배수효과, +상승/−하락) | "이 위치가 비싼/싼 이유" |
| `신뢰도` | 상 / 중 / 하 (학습데이터 근접도) | '하'면 "참고용" 경고 표시 |
| `최근접학습정류장_m` | 외삽 정도(클수록 불확실) | 신뢰도 보조 |
| `자치구_근사` | 근사 자치구 | 라벨·필터 |

- **조회법**: 클릭 좌표 → 격자에서 최근접 점(약 250m 이내). **250m 안에 칸이 없으면** "이 위치엔 정류장/예측 없음"으로 처리.
- CSV를 그대로 fetch해도 되고, 빌드 때 **JSON/GeoJSON으로 변환**해 써도 됩니다(자유).
- `grid_250m.csv`는 일반 git 파일(1.4MB)이라 **LFS pull 없이도** 바로 씁니다.

## 기술 스택 = 담당 재량
지도 라이브러리(Leaflet / MapLibre / …), 프레임워크, 빌드도구 모두 자유. 단 **모드 A이므로 "정적 빌드 산출물"** 이 나오게 해주세요(배포가 쉬움). 비밀키·환경변수 필요 없음.

## 배포 (저장소 오너가 진행: GitHub Actions + Cloudflare Pages 예정)
- **정적 사이트**로 빌드(서버 런타임 의존 X).
- 모노레포이므로 Cloudflare Pages의 **루트/빌드 디렉터리를 `web/`** 로 지정해 배포합니다.
- 배포 산출물이 **자기완결적**이도록: 빌드 시 `../model/grid_250m.csv`(또는 변환한 JSON)를 **web 자산으로 복사·번들**(런타임에 repo 바깥 의존·CORS 회피).

## 시작하기
1. repo clone (격자만 쓰면 `git lfs pull` 불필요).
2. `web/`에서 본인 스택으로 스캐폴드.
3. **브랜치 파서 작업 → PR**(같은 repo, `model/` 팀과 충돌 방지).
